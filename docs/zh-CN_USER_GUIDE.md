# Kruise Rollout 分批发布扩展使用指南

## 一、入口位置

登录 Rancher 后，进入目标集群，在左侧导航栏找到 **"分批发布"** 菜单。

点击后进入扩展主界面，显示两个资源类型：
- **Rollout** - 发布策略配置
- **BatchRelease** - 批次发布控制

---

## 二、查看 Rollout（发布策略）

### 2.1 列表页面

进入 **分批发布 → Rollout**，查看所有发布策略：

| 列名 | 说明 |
|------|------|
| 状态 | 当前发布状态（运行中/已完成/失败） |
| 名称 | Rollout 资源名称 |
| 阶段 | 当前所处发布阶段 |
| 灰度步骤 | 已完成/总步骤数 |
| 消息 | 状态描述信息 |
| 创建时间 | 资源创建时间 |

### 2.2 查看详情

点击任意 Rollout 名称，进入详情页查看：

- **概览**: 基本信息、关联工作负载、当前状态
- **关联资源**: 关联的 BatchRelease
- **事件**: 发布过程中的事件日志
- **YAML**: 完整配置

### 2.3 状态说明

| 状态 | 含义 |
|------|------|
| ✨ Progressing | 发布进行中 |
| ✅ Healthy | 发布正常完成 |
| ⚠️  Degraded | 发布失败或异常 |
| ⏸️  Paused | 发布已暂停 |

---

## 三、查看 BatchRelease（批次发布）

### 3.1 列表页面

进入 **分批发布 → BatchRelease**，查看所有批次发布记录：

| 列名 | 说明 |
|------|------|
| 状态 | 当前批次状态 |
| 名称 | BatchRelease 名称 |
| 工作负载 | 关联的工作负载类型 |
| 阶段 | 发布阶段 |
| 当前批次 | 当前执行到的批次 |
| 批次状态 | 当前批次的执行状态 |

### 3.2 查看详情

点击 BatchRelease 名称，查看：
- 目标工作负载信息
- 发布计划详情
- 各批次执行进度
- 历史事件

---

## 四、在工作负载页面查看关联发布

### 4.1 支持的工作负载类型

在以下工作负载的详情页会自动显示"分批发布"标签页：

- Deployment
- StatefulSet
- DaemonSet
- Job
- CronJob

### 4.2 操作步骤

1. 进入 **集群 → 工作负载**
2. 选择任意受支持的工作负载类型（如 Deployment）
3. 点击目标工作负载名称进入详情页
4. 切换到 **"分批发布"** 标签页

### 4.3 标签页内容

在标签页中可以看到：
- 关联的 **Rollout** 列表
- 关联的 **BatchRelease** 列表
- 点击资源名称可跳转到对应详情页

---

## 五、常见操作

### 5.1 筛选和搜索

- **按命名空间筛选**: 使用顶部命名空间下拉框
- **按状态筛选**: 点击状态图标进行筛选
- **搜索**: 使用顶部搜索框输入名称

### 5.2 刷新列表

点击列表顶部的 **"刷新"** 按钮更新数据。

### 5.3 YAML 编辑

在 Rollout/BatchRelease 详情页，点击 **"编辑 YAML"** 可直接修改配置。

### 5.4 删除资源

删除 Rollout 或 BatchRelease 资源：

1. **进入列表页面**
   - 进入 **分批发布 → Rollout** 或 **分批发布 → BatchRelease**

2. **执行删除**
   - 方式一：在列表中找到目标资源，点击右侧 **⋮** 菜单，选择 **删除**
   - 方式二：点击资源名称进入详情页，点击页面右上角的 **⋮** 菜单，选择 **删除**

3. **确认删除**
   - 系统会弹出确认对话框
   - 输入资源名称进行二次确认
   - 点击 **删除** 按钮完成操作

### 5.5 注意事项

- **删除 Rollout 会同时删除关联的 BatchRelease**
- **正在进行的发布无法直接删除**，需先暂停或等待发布完成
- 删除后无法恢复，请谨慎操作

## 六、故障排查

### 6.1 看不到"分批发布"菜单

1. 确认 Rancher 版本 >= 2.10.0
2. 确认 OpenKruise 已安装并正常运行
3. 刷新浏览器页面

### 6.2 工作负载详情页没有标签页

1. 确认工作负载类型是支持的类型
2. 确认已有对应的 Rollout 或 BatchRelease 资源
3. 检查资源是否在同一命名空间

### 6.3 发布卡住不动

1. 查看详情页的 **事件** 标签页
2. 检查关联工作负载状态
3. 确认 Rollout 配置是否正确
